Бортовий Комп'ютер (On-Board Computer) на базі Raspberry Pi Pico

Автор: tor4man (GitHub: tor4man66)
Платформа: Raspberry Pi Pico (RP2040)
Середовище: Arduino IDE з FreeRTOS
Дисплей: OLED 128x128 (SH1107, I2C)
Протокол діагностики: KWP1281 (K-Line) для VAG автомобілів (Audi 80 та інші)

Бібліотеки:
KLineKWP1281Lib, 
FreeRTOS, 
Adafruit_SH110X, 
Adafruit_GFX_Library, 
U8g2 (Оригінал),
Adafruit_BusIO (Оригінал).

📋 Можливості

Основні функції

· Миттєва витрата палива (л/год та л/100км) з розрахунком через імпульси форсунки
· Швидкість автомобіля (км/год) за датчиком VSS
· Оберти двигуна (RPM) за сигналом запалювання
· Напруга бортової мережі з корекцією мертвого часу форсунки
· Рівень палива (літри та відсотки) з медіанним фільтром
· Статистика поїздок (TRIP: витрата, відстань; PERS: довготривала)
· Регулювання яскравості OLED

Діагностика K-Line (KWP1281)

· Підключення до ЕБУ двигуна (5-бод ініціалізація, 4800 бод)
· Читання кодів помилок (DTC) — до 5 одночасно на екрані
· Скидання помилок (Clear Faults) по довгому натисканню кнопки
· Індикатор Check Engine — блимає при помилках зв'язку
· Сервісний режим — відображення даних з груп вимірювань K-Line

Режими сну

· Dormant Mode — глибокий сон RP2040 з пробудженням по ACC
· Споживання в сні: < 0.1 мА (повне вимкнення PLL, XOSC, периферії)
· Пробудження по передньому фронту сигналу ACC (GPIO 28)

Датчики безпеки

· Рівень гальмівної рідини
· Тиск мастила (низький 0.3 бар + високий 1.8 бар)
· Перегрів / низький рівень охолоджувальної рідини
· Контроль зарядки генератора

---

🎛️ Керування

·Перемикання екранів (0→1→2) Коротке натискання кнопки
·Скидання TRIP Довге натискання в режимі 0
·Скидання помилок ЕБУ Довге натискання в режимі 2
·Вхід у сервісний режим Утримання кнопки при ACC ON

---

🖥️ Екрани

· Режим 0: Миттєва витрата + статистика TRIP/PERS
· Режим 1: Швидкість, оберти, напруга, рівень палива, час впорскування
· Режим 2: Коди помилок K-Line (DTC) з можливістю скидання
· Сервісний режим: Дані з груп вимірювань K-Line

📂 Структура проекту


```
├── main.ino                       # Головний файл (setup, loop, ініціалізація)
├── Config.h                       # Всі налаштування (піни, таймінги, калібрування, тексти)
│
├── StateManager.h/.cpp            # Singleton-сховище стану всієї системи
├── HardwareManager.h/.cpp         # Робота з GPIO, ADC, OLED, I2C, WDT, сон
├── SystemInitializer.h/.cpp       # Ініціалізація всіх компонентів у правильному порядку
│
├── KLineManager.h/.cpp            # Діагностика KWP1281 (з'єднання, DTC, скидання помилок)
├── KLineTypes.h                   # Enum KLineConnectionState (DISCONNECTED, CONNECTING, CONNECTED...)
│
├── DisplayRenderer.h/.cpp         # Оркестратор рендерингу екранів
├── DisplayComponents.h/.cpp       # Підготовка даних для всіх режимів (Mode 0/1/2/Service)
├── ScreenRenderers.h/.cpp         # Функції малювання екранів
├── DisplayStateMachine.h/.cpp     # Машина станів дисплея 
├── DisplayExtensions.h/.cpp       # Допоміжні функції малювання 
│
├── ServiceModeManager.h/.cpp      # Менеджер сервісного режиму (K-Line групи вимірювань)
├── DormantManager.h/.c            # Глибокий сон (Dormant Mode) для RP2040
│
├── CalculationEngine.h/.cpp       # Розрахунок витрати, відстані, швидкості, TRIP/PERS
├── PowerMonitor.h/.cpp            # Моніторинг напруги бортової мережі та стану ACC
├── FuelMonitor.h/.cpp             # Моніторинг рівня палива (ADC + згладжування)
├── EngineMonitor.h/.cpp           # Моніторинг RPM та стану двигуна
├── BrightnessMonitor.h/.cpp       # Моніторинг  яскравості OLED
│
├── ErrorCoordinator.h/.cpp        # Координація помилок (збір, пріоритезація, сирена)
├── SensorFunctions.h/.cpp         # Перевірка цифрових датчиків (гальма, тиск, перегрів)
├── ButtonController.h/.cpp        # Обробка кнопки (коротке/довге натискання)
├── AudioManager.h/.cpp            # Звукові сповіщення (зумер, частоти, тривалості)
├── DataPersistence.h/.cpp         # Збереження та завантаження даних з EEPROM
│
├── IRQHandlers.h/.cpp             # Обробники переривань (форсунка, VSS, RPM)
├── GlobalIRQState.h               # Глобальний стан лічильників переривань
│
├── Icons.h                        # Іконки помилок (бітмапи, анімації, координати)
├── ForwardDeclarations.h          # Forward declarations для уникнення циклічних залежностей
├── DebugUtils.h/.cpp              # Логування (ініціалізація, debug, error logger)
```
